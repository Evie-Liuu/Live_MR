# 恢復開發模式（去 Docker 版）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在完全不依賴 Docker 的前提下，恢復前後端各自的熱重載開發流程——新增一支輕量 backend dev entrypoint 與一支獨立跑原生 `livekit-server.exe`（單機模式，只綁 localhost）的 dev 腳本，讓 `frontend` 既有的 `npm run dev`（Vite HMR）能正常運作。

**Architecture:** 三個獨立終端機視窗、各自獨立重啟：`npx tsx scripts/dev-livekit.ts`（LiveKit 子行程，重用既有的 `launcher/livekitConfig.ts`/`livekitProcess.ts`，只綁 `127.0.0.1:7880`）、`cd backend && npm run dev`（新的 `src/dev.ts`，純 HTTP `:3001`，`tsx watch` 熱重載）、`cd frontend && npm run dev`（Vite，`:5173`，HMR，proxy 設定不用改）。

**Tech Stack:** 沿用既有 TypeScript/Express/tsx，`scripts/dev-livekit.ts` 需要在根目錄新增 `tsx` devDependency 才能執行（純 `node` 無法執行 `.ts`）。

## Global Constraints

- Dev 模式的 LiveKit 只綁 `127.0.0.1`，不支援其他裝置連線（那個用途用既有的 `build-launcher.mjs` + `LiveMR.bat` 流程測，不在這裡重做）。
- `frontend/vite.config.ts` 不要改動——它現有的 proxy 設定（`/api`→`localhost:3001`、`/livekit`→`ws://localhost:7880`）已經對得上這次新增的兩個 port。
- 不做 TLS——`localhost` 對 `getUserMedia` 視為 secure context。
- `scripts/dev-livekit.ts` 直接重用 `backend/src/launcher/livekitConfig.ts`/`livekitProcess.ts`，不得複製貼上重寫一份。
- Commit 訊息不要加 `Co-Authored-By` 字樣。
- 這兩個新檔案都是「跑起來看有沒有正常動作」的整合型腳本（類似既有的 `standalone.ts`/build 腳本），比照本專案慣例不強求 unit test，改用手動驗證把關。

- [ ] **Step 1: 安裝 tsx**

Run: `cd /c/Project/Live_MR && NODE_OPTIONS=--use-system-ca npm install --save-dev tsx`（若遇到 TLS 憑證錯誤，這是本機 Avast 攔截的已知問題，`NODE_OPTIONS=--use-system-ca` 這個環境變數就是為此而加）

- [ ] **Step 2: 建立 `scripts/dev-livekit.ts`**

```typescript
import fs from 'node:fs'
import path from 'node:path'
import { fileURLToPath } from 'node:url'
import { buildLivekitConfig } from '../backend/src/launcher/livekitConfig.js'
import { LiveKitProcess } from '../backend/src/launcher/livekitProcess.js'

const __dirname = path.dirname(fileURLToPath(import.meta.url))
const ROOT = path.resolve(__dirname, '..')
const LIVEKIT_BIN = path.join(ROOT, '.build-cache', 'livekit-server.exe')

async function main(): Promise<void> {
  if (!fs.existsSync(LIVEKIT_BIN)) {
    console.error(`找不到 ${LIVEKIT_BIN}。請先執行：node scripts/fetch-livekit-server.mjs`)
    process.exit(1)
  }

  const configYaml = buildLivekitConfig({
    nodeIp: '127.0.0.1',
    apiKey: 'devkey',
    apiSecret: 'devsecret1234567890devsecret1234567890',
  })

  const livekit = new LiveKitProcess()
  await livekit.start({
    binPath: LIVEKIT_BIN,
    configYaml,
    workDir: path.join(ROOT, '.build-cache', 'livekit-dev-data'),
    port: 7880,
  })

  console.log('LiveKit（dev，僅 localhost）已啟動：ws://127.0.0.1:7880\n按 Ctrl+C 結束。')

  const shutdown = async (): Promise<void> => {
    console.log('\n正在關閉 LiveKit...')
    await livekit.stop()
    process.exit(0)
  }
  process.on('SIGINT', shutdown)
  process.on('SIGTERM', shutdown)
}

main().catch((err) => {
  console.error('啟動失敗：', err)
  process.exit(1)
})
```

### Task 2: `backend/src/dev.ts`（輕量 dev entrypoint）

**Files:**
- Create: `backend/src/dev.ts`
- Modify: `backend/package.json`（加回 `dev` script）

**Interfaces:**
- Consumes: 既有的 `RoomStore`（`rooms.ts`）、`createRouter`（`routes.ts`）、`RecordingStore`（`recording.ts`）、`RoomAdminService`（`roomAdmin.ts`）——四者的建構/呼叫方式與 `standalone.ts` 一致，不變更任何既有介面。

- [ ] **Step 1: 建立 `backend/src/dev.ts`**

```typescript
import express from 'express'
import cors from 'cors'
import { RoomStore } from './rooms.js'
import { createRouter } from './routes.js'
import { RecordingStore } from './recording.js'
import { RoomAdminService } from './roomAdmin.js'

// 對應 scripts/dev-livekit.ts 起的 dev 用 LiveKit（僅 localhost，devkey/devsecret 為固定值）。
process.env.LIVEKIT_URL = 'ws://127.0.0.1:7880'
process.env.LIVEKIT_API_KEY = 'devkey'
process.env.LIVEKIT_API_SECRET = 'devsecret1234567890devsecret1234567890'

const app = express()
app.disable('x-powered-by')
// Vite dev server（:5173）用 proxy 轉發 /api，瀏覽器本身視角是同源請求；
// 這裡的 CORS 設定只在有人直接對 :3001 發請求（例如用 curl/Postman 除錯）時才用得到。
app.use(cors({ origin: 'http://localhost:5173' }))
app.use(express.json({ limit: '25mb' }))

const store = new RoomStore()
const recordingStore = new RecordingStore()
const roomAdmin = new RoomAdminService()

app.get('/api/health', (_req, res) => res.json({ status: 'ok' }))
app.use('/api', createRouter(store, { recordingStore, roomAdmin }))

const CLEANUP_INTERVAL = 5 * 60 * 1000
const ROOM_TTL = 2 * 60 * 60 * 1000
setInterval(() => store.cleanup(ROOM_TTL), CLEANUP_INTERVAL)

const PORT = 3001
app.listen(PORT, () => {
  console.log(`Dev backend listening on http://localhost:${PORT}（配合 frontend 的 Vite dev server 使用，不要直接打開這個網址）`)
})
```
