# Live MR 使用說明

> 對象：**使用本產品的老師與學生**（非技術文件）。
> 技術架構請見 [`ARCHITECTURE.md`](./ARCHITECTURE.md)；錄製細節見 [`recording-flow.md`](./recording-flow.md)。

---

## 這是什麼產品？

**Live MR** 是一套**即時混合實境（MR）英語會話教學平台**。

老師與學生各自打開攝影機，系統會即時把每個人的動作（身體、表情、手勢）套用到一個專屬的 **3D 虛擬角色（VRM）** 身上，所有角色一起出現在教室「**大屏**」投影的共享場景裡——就像大家一起站進同一間虛擬服飾店，面對面對話練習英文。

老師說話時，內建的 **AI 助理**會即時聽懂內容，自動幫學生生成「該怎麼回答老師」的英語範例句，讓學生不再對著鏡頭愣住。整堂課還能一鍵**錄製**下來，下課後回放或存檔。

一句話：**大家一起變成虛擬人物，AI 隨堂幫學生想台詞，全程可錄影回顧。**

## AI 特色（本產品的核心亮點）

傳統教學提示卡只能寫死固定句子；Live MR 的 AI 助理是**即時、聽得懂上下文、會自己補完細節**的教學助手，設計重點如下：

### 1. AI 直接「聽懂」老師說的英文（原生語音理解）

老師說話時，系統會錄下這段語音，**直接送給 Google Gemini 模型聽**（而不是先轉成不精確的逐字稿再猜）。這代表：
- 老師講話快、口音重、或中英夾雜，AI 仍能盡量聽懂並修復常見的誤判（例如價格、尺寸、人名這類容易聽錯的詞）。
- 螢幕上同步顯示老師說話的文字稿，方便老師確認 AI 有沒有聽對。

### 2. 自動從一長串話裡「抓出真正的問題」

老師上課常常是一長串話：問候 + 指令 + 閒聊 + 才問到重點。AI 會先在內部判斷「老師這段話裡，哪一句才是真正要問學生的問題」，再針對那句話生成回答建議，不會被前面的寒暄干擾。AI 能辨識的問句類型很廣，包括：
- Yes/No 問句、What/Who/Where/When/Why/How
- 二選一（「你喜歡飯還是麵？」）
- 過去經驗、未來計畫、個人偏好、意見
- 甚至是「命令句」形式的邀請發言（例如「跟我說說你最喜歡的動物」）

### 3. 三種難度的提示，因材施教

每次生成，AI 會準備三種版本，老師可依學生程度切換：

| 難度 | 說明 | 適合對象 |
|---|---|---|
| 🔀 **重組** | 一句完整答句，但單字順序被打散成一顆顆字卡，學生要自己排回正確順序 | 需要練習句型結構的學生（**預設模式**） |
| ✅ **完整** | 一句最簡單、可以直接照著唸出來的完整英語回答 | 初學或需要立即開口的學生 |
| ➕ **延伸** | 在完整答句後面加一句更豐富的延伸內容（補充資訊或反問） | 程度較好、想多練一點的學生 |

### 4. AI 會「自己編故事」，不會卡住教學

如果老師問到一個場景裡沒設定過的細節（例如「這件外套多少錢？」但系統本來沒寫定價格），AI 不會回答「我不知道」讓對話卡住——它會**自己編一個合理、具體的答案**（例如「200 元」），並且記住這個設定，讓同一堂課接下來的對話維持一致（同一件外套之後問起，價格不會變來變去）。

### 5. 貼著「這堂課的教學目標」走

老師目前選的場景與任務（例如：「詢問一件藍色 T 恤的價格」）會被一併告訴 AI，讓生成的答案盡量貼近這堂課要練習的句型與學生扮演的角色（顧客 / 店員）。但如果老師臨時聊到別的話題，AI 也會自然回答，不會硬把對話拉回課程進度。

### 6. 老師掌控節奏的「輪流互動」模式

AI 助理面板提供一個簡單的「開始互動」流程，老師完全掌控步調：

```
[開始互動]  → 老師說話（自動錄音）
     ↓
[換學生]    → AI 生成提示 → 廣播給學生 / 大屏
     ↓
輪到學生看提示回答（可自行再要求「延伸」一句）
     ↓
[輪到自己]  → 換老師接著往下問 → 重複
     ↓
[結束互動]
```

老師按下「換學生」的瞬間，系統會自動送出剛剛那段語音給 AI 生成提示，並**同時**廣播到：
- 每位學生自己的畫面（右上角提示卡片，可縮放/拖曳/最小化）
- 教室大屏（底部提示橫幅，全班共同看得到）

如果自動倒數（3 秒）內老師不做任何操作，系統預設會自動用「重組」模式廣播；老師隨時可以手動按「完整」或「延伸」覆寫。

### 7. 隱私與安全

AI 使用的金鑰只存在後端伺服器，瀏覽器（老師 / 學生端）永遠不會拿到金鑰；老師的語音只在生成提示當下短暫使用，不會被存成教學紀錄的一部分（僅錄製功能會另外存下課堂音訊，見下方「課堂錄製」）。

## 老師：如何開始一堂課

1. **建立房間**：進入首頁，選擇「老師」角色，系統會建立一個新房間，並顯示 QR code 與房間代碼。
2. **等待學生加入**：學生掃碼或輸入房間代碼申請加入，老師在等待畫面按「允許」或「拒絕」。
3. **進入課堂**：按下開始後進入 `HostSession`，可以看到：
   - 自己的攝影機畫面與虛擬角色預覽
   - 已加入的學生列表
   - 場景 / 任務切換面板
   - 「🤖 AI 助理」分頁
4. **選擇場景與任務**：從場景清單挑選主題（例如「服飾店 → 收銀台」），指派學生扮演的角色（顧客 / 店員），並選定目前要練習的任務（例如「詢問藍色 T 恤的價格」）。
5. **開啟大屏**：另開一個瀏覽器分頁 / 投影到教室螢幕，網址加上 `?screen=bigscreen`，即可看到所有人同框的 3D 場景。
6. **使用 AI 助理**：切到「AI 助理」分頁，按「開始互動」開始錄音提問，講完按「換學生」讓 AI 生成提示（詳見上方「AI 特色」）。
7. **錄製課堂（選用）**：按「▶ 開始錄製」，下課按「⏹ 停止錄製」，系統會在背景自動把大屏畫面與所有人聲音合成一支 MP4，稍後可在錄製清單下載。
8. **管理課堂**：可隨時靜音或請出特定學生。

## 學生：如何加入一堂課

1. 用手機或電腦掃老師畫面上的 **QR code**，或直接輸入 **房間代碼** / 開啟連結。
2. 輸入自己的名字，送出加入申請。
3. 等待老師「允許」加入（畫面會顯示等待狀態）。
4. 允許後進入課堂，授權瀏覽器使用攝影機 / 麥克風，即可看到自己的虛擬角色。
5. 老師進行互動時，畫面右上角會跳出「**輪到你說話了**」提示卡片，依難度顯示：
   - 重組模式：一格一格的英文單字卡，自己在腦中（或口說）排出正確順序
   - 完整 / 延伸模式：整句可直接朗讀的英文
   - 可展開「老師原話」對照老師剛剛說了什麼
   - 也可以自己按「延伸」再要一句更豐富的補充句
6. 說完後等待老師換下一輪提問即可，不需要任何操作。

## 課堂錄製

- 錄製涵蓋：教室大屏畫面 + 老師與每位學生的聲音，停止後自動合成成單一 MP4。
- 錄製期間**大屏視窗必須保持開啟**，中途關閉會中斷該次錄製。
- 建議單次錄製不超過 10 分鐘。
- 詳細技術流程見 [`recording-flow.md`](./recording-flow.md)。

## 系統需求

- **瀏覽器**：建議使用 **Chrome** 或 **Edge**（AI 助理的語音辨識/擷取功能依賴 Web Speech API，Firefox / Safari 目前不支援，AI 助理面板會顯示警示並停用錄音）。
- **裝置**：需要攝影機（動作捕捉）與麥克風（AI 助理、錄音）。
- **網路**：需要能連上教室 Wi-Fi / 網路，建議連線穩定（AI 生成、多人視訊皆走即時網路）。
- **連線方式**：需以 **HTTPS** 開啟（相機權限的瀏覽器限制），請使用老師提供的正式網址，而非 `http://`。
- **老師端啟動**：解壓縮後雙擊 `LiveMR.bat`，免安裝其他軟體。

## 常見問題（FAQ）

**Q: AI 提示多久會出來？**
老師按下「換學生」後，通常數秒內（視網路與 AI 服務忙碌程度）就會生成，畫面會顯示「AI 生成中…」。

**Q: 如果 AI 沒聽懂老師說的話怎麼辦？**
可以直接重講一次，或用手動模式（完整 / 延伸按鈕）重新觸發生成；畫面上也會同步顯示辨識出的文字稿方便確認。

**Q: 學生可以自己跟 AI 對話嗎？**
目前設計是「老師主導提問，AI 幫學生想怎麼回答」，學生端只能針對已收到的提示再要求一句「延伸」，不能自由對話。

**Q: 我的瀏覽器沒有攝影機權限怎麼辦？**
請確認瀏覽器網址列有允許「攝影機」「麥克風」權限，並確認是用 HTTPS 網址開啟。

**Q: 影片錄好在哪裡下載？**
課堂結束後，回到老師端的錄製清單，選擇對應的場次即可下載合成好的 MP4（合成需要一點時間，狀態會顯示「合成中」到「完成」）。

## 已知限制

- AI 助理需要瀏覽器支援語音辨識（建議 Chrome / Edge）。
- 課堂資料（房間、錄製進度）保存在伺服器記憶體中，若伺服器重新啟動，進行中的房間與錄製對照會遺失（已產生的影音檔案仍在磁碟上）。
- 目前僅有「服飾店・收銀台」場景已完整上線，更多場景陸續增加中。
